# examples — демонстрационные песочницы

Примеры разнесены по режимам. В каждой секции — свой `README.md` с инструкцией
запуска и таблицей ожидаемых результатов.

- [`normalizer/`](normalizer/) — дерево фикстур для нормализатора (`fs-normalizer`):
  по подпапке на правило, плюс кейсы конвейера, фильтра `.fs-ignore` и крайних
  сценариев. Канонический запуск — `--dry-run` (без изменений). Боевой прогон
  **меняет** фикстуры на месте, поэтому рядом лежат скрипты отката
  `reset.sh`/`reset.command`/`reset.bat` (возвращают дерево к версии из git).
- [`checker/`](checker/) — дерево-песочница для проверки структуры (`fs-checker`)
  с файлом правил `.fs-check`. Проверка структуру **не меняет**, поэтому скриптов
  отката здесь нет.
- [`syncher/`](syncher/) — песочница синхронизации (`fs-syncher`) с `.fs-sync.toml`,
  источником и локальными каталогами-приёмниками. Прогон **без сети**; канонический
  запуск — `--dry-run` (без изменений). Боевой прогон мутирует песочницу — откат через
  `git restore`/`git clean` (отдельных `reset.*` нет).

Запускать примеры удобно прямо из этих каталогов (см. README секции): командами
`fs-normalizer`/`fs-checker`/`fs-syncher` или обёртками из [`bin/`](../bin/).
