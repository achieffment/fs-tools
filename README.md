# fs-normalizer

CLI-утилита для рекурсивной нормализации имён файлов и папок: транслитерация в ASCII, приведение дат к ISO-формату, единый стиль разделителей и регистра. Поведение идемпотентно — повторный запуск над уже нормализованным деревом ничего не меняет.

## Возможности

- Рекурсивный обход каталога с переименованием файлов и папок.
- Транслитерация не-ASCII символов (кириллица, умляуты, emoji и т.п.) в ASCII.
- Распознавание дат в разных форматах и приведение к ISO (`YYYY-MM-DD`) с плейсхолдерами `?` для недостающих компонентов.
- Ведущий ноль для однозначных числовых токенов (`1_file` → `01_file`).
- Пробелы → дефис, обрезка «мусора» по краям имени, единый регистр (папки — с заглавной, файлы — в нижнем).
- Расширение файла не изменяется.
- Скрытые файлы и папки (начинающиеся с `.`) пропускаются и не обходятся.
- Корневой каталог не переименовывается.
- Безопасная обработка конфликтов: если целевое имя уже занято другим объектом, переименование пропускается (данные не перезаписываются).
- Выбор каталога: нативный проводник Windows (на Windows и в WSL), стандартный диалог macOS либо ввод пути в терминале (на обычном Linux).

## Требования

- Python 3.10+ (используется синтаксис `X | None`).
- [Unidecode](https://pypi.org/project/Unidecode/) — единственная рантайм-зависимость.

Стандартной библиотеки достаточно для выбора папки: на Windows/WSL вызывается `powershell.exe` (нативный диалог), на macOS — `osascript` (AppleScript `choose folder`), на обычном Linux — ввод пути в терминале. Дополнительных GUI-пакетов не требуется.

## Использование

Запуск без аргументов — каталог выбирается интерактивно:

```bash
python3 normalize_fs.py
```

Поведение в зависимости от среды:

- **Windows** — открывается нативный диалог выбора папки (проводник).
- **WSL** — открывается нативный диалог Windows, выбранный путь автоматически переводится в путь WSL (`/mnt/...`).
- **macOS** — открывается стандартный системный диалог `choose folder` (через `osascript`), возвращается POSIX-путь.
- **Обычный Linux** — запрашивается ввод пути к каталогу в терминале.

Тонкие обёртки (просто пробрасывают вызов):

```bash
./normalize.sh        # Linux/macOS
normalize.bat         # Windows
```

После завершения выводится сводка: сколько объектов переименовано и сколько пропущено.

## Правила нормализации

Правила применяются к имени (без расширения для файлов) строго в указанном порядке:

| # | Правило | Что делает |
|---|---------|------------|
| 1 | `TransliterationRule` | Любой не-ASCII символ → ASCII |
| 2 | `DateRule` | Даты → ISO `YYYY-MM-DD`; недостающие части → `??` |
| 3 | `LeadingZeroRule` | Однозначный числовой токен → с ведущим нулём |
| 4 | `SpaceToDashRule` | Пробелы (и их повторы) → одиночный дефис |
| 5 | `TrimEdgeRule` | Обрезка не буквенно-цифровых символов по краям (с сохранением `?`) |
| 6 | `CaseRule` | Папки — с заглавной буквы, файлы — в нижнем регистре |

Порядок важен: `CaseRule` идёт последним (после схлопывания пробелов и обрезки кромок), что обеспечивает корректную капитализацию за один проход и идемпотентность.

### Примеры

Файлы:

| Исходное имя | Результат |
|--------------|-----------|
| `Отчёт.TXT` | `otchiot.TXT` |
| `1_file.TXT` | `01_file.TXT` |
| `v2 readme.MD` | `v2-readme.MD` |
| `20.05.2020_dump` | `2020-05-20_dump` |
| `05.2020_report` | `2020-05-??_report` |
| `2020` | `2020-??-??` |
| `-file_01-.png` | `file_01.png` |

Папки:

| Исходное имя | Результат |
|--------------|-----------|
| `отчёт за март` | `Otchiot-za-mart` |
| `Отчёт 2020` | `Otchiot_2020-??-??` |
| `  отчёт` | `Otchiot` |

## Публичное API

Импортируйте из пакета `normalizer`, а не из подмодулей напрямую:

```python
from normalizer import build_normalizer, FilesystemNormalizer

normalizer = build_normalizer()
normalizer.normalize("Отчёт 2020", is_dir=True)   # 'Otchiot_2020-??-??'

fs = FilesystemNormalizer(build_normalizer())
renamed, skipped = fs.apply(Path("/path/to/dir"))
```

Экспортируется: `main`, `FilesystemNormalizer`, `NameNormalizer`, `build_normalizer`, `Rule`, `TransliterationRule`, `DateRule`, `LeadingZeroRule`, `CaseRule`, `SpaceToDashRule`, `TrimEdgeRule`.
