# Примеры — по режимам и кейсам

> Claude-эквивалент [`.cursor/rules/examples.mdc`](../../.cursor/rules/examples.mdc). Применяется при работе с `examples/**`.

Примеры разнесены по режимам: `examples/normalizer/`, `examples/checker/`,
`examples/syncher/` и `examples/schemer/`. Каждая
секция самодостаточна (свой `README.md`); корневой `examples/README.md` — индекс.

## normalizer

Фикстуры в `examples/normalizer/` сгруппированы по правилам: `NN-<rule>/<case>/`, где
`NN` — порядковый номер правила, `<case>` — отдельный кейс (`digit`, `text`,
`unmatched`). Папка `mix` **всегда последняя**; новое правило добавляется перед ней,
`mix` перенумеровывается (сейчас `09-brackets`, `10-ignore`, `11-mix`).

Демо-файл `examples/normalizer/.fs-ignore` (секция `10-ignore`) читается из самого
нормализуемого каталога; **все** его паттерны привязаны к `10-ignore`, поэтому
таблицы `01`–`09` и `11-mix` от него не зависят. Проверки идемпотентности/конфликтов
фильтр **снимают** (`rm -f .fs-ignore`).

Для каждого кейса — **две** формы примера: без расширения (`Файл (1`) и с
расширением в нижнем регистре (`Файл (1.txt`). Исключение — кейсы фильтра `10-ignore`
(они про сопоставление путей, повторяют реальные цели паттернов).

Нормализатор **меняет** фикстуры на месте, поэтому рядом лежат `reset.sh`/
`reset.command`/`reset.bat` (откат к версии из git: `reset` → `clean` → `checkout`,
все с `core.ignorecase=false`, плюс явное удаление `.fs-log`). Скрипты работают
только внутри своего каталога (pathspec `.`).

### Уникальность нормализованных имён в папке-кейсе

Внутри одной `<case>/` все фикстуры должны давать **различные** нормализованные имена
(две формы одного примера не в счёт — их разводит расширение). Иначе на каждом
прогоне сыплется ложный `Пропуск (конфликт)`, а «выживший» файл зависит от платформы.
**Единственное намеренное исключение** — `08-edge-cases/conflict/` (`a b.txt` +
`a-b.txt`). Коллизии устраняй точечно, без массовых переименований.

## checker

`examples/checker/` — дерево-песочница с `.fs-check`. Проверка структуру **не меняет**,
reset-скриптов здесь нет. Пустые каталоги git не хранит, поэтому в листовых папках
лежит `.gitkeep` (скрыт, на проверку не влияет). Часть путей **намеренно отсутствует**,
чтобы прогон показывал нарушения и по папкам, и по файлам, а ветка `_Archive`
демонстрировала исключение через ordered pathspec-негативы.

## syncher

`examples/syncher/` — песочница синхронизации: `.fs-sync.toml` (профили `[[sync]]`
«site» и `[[backup]]` «vault»), источник `source/`/`archive/` и **локальные**
каталоги-приёмники `server/`/`server-archive/` (`remote_root` указывает внутрь
песочницы) — прогон **без сети** и детерминирован. Канонический запуск — `--dry-run`
(без мутаций), его итог зафиксирован в `README.md` секции и проверяется `test_examples`.
Production-прогон мутирует песочницу — откат через `git restore`/`git clean` (отдельных
`reset.*` нет). Пустой приёмник держит `.gitkeep` (это каталог-приёмник, на
синхронизацию не влияет). Меняя профили/правила — обнови таблицу и ожидаемый итог в
`README.md` секции.

## schemer

`examples/schemer/Warehouse/` — проверяемый корень (там же `fs-schm.toml`); сам
`README.md` секции лежит **вне** этого корня, чтобы не попадать под F15
(loose-файл). Дерево — один тематический узел со всеми группами конфига и
намеренными нарушениями по одному кейсу на каждую категорию F1–F15 (обязательный
файл отсутствует, опциональный с плохим заголовком, пустая группа, файл вне
групповой папки). Проверка **не меняет** дерево, reset-скриптов нет. Пустая группа
держит `.gitkeep` (скрыт, не снимает пометку «пусто»).

Новый/изменённый кейс отражай в таблице соответствующей секции `README.md`.
