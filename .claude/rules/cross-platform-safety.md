# Кросс-платформенная безопасность

> Claude-эквивалент [`.cursor/rules/cross-platform-safety.mdc`](../../.cursor/rules/cross-platform-safety.mdc). Применяется при работе с `normalizer`/`syncher`/`shared/picker.py`/`shared/env.py`/`shared/log.py`.

Код безопасно исполняется на Windows, WSL, Linux и macOS. Файловые операции не
должны приводить к потере файлов и данных.

- **Не выходить за рамки**: нормализатор переименовывает только содержимое выбранного
  каталога; сам корень не трогаем, по скрытым (`.`) объектам не заходим. Checker
  структуру не меняет вовсе (единственная запись — `.fs-log.log`).
- **Имя — один компонент пути**: разделители (`/`, `\`), управляющие и запрещённые
  на Windows символы (`< > : " | ? *`) не должны попадать в `os.rename`. Чистит
  `TransliterationRule`, страховочный барьер — `normalizer/safety.py`
  (`enforce_safe_component`), вызывается в `name.py`. Не ослабляй их.
- **Без перезаписи**: занятый другим объектом `dest` — конфликт, элемент пропускаем;
  конфликт учитывается в `FsNormalizer.conflicts` (безопасный пропуск) и на
  код возврата НЕ влияет.
- **Порядок**: переименование deepest-first (дети раньше родителей).
- **Case-only** переименование на регистронезависимых ФС — через временное имя.
- **Журнал `.fs-log.log`** (`shared/log.py`, `append_log`) пишется в корень выбранного
  каталога только режимом append (`open("a")`, `utf-8`) и под скрытым именем — он
  ничего не перезаписывает и сам обходом пропускается.
- **Выбор каталога** (`shared/picker.py`): нативный диалог Windows/WSL
  (`IFileOpenDialog` через `powershell.exe` + `pick_folder.ps1`, `wslpath`, проброс
  через `WSLENV`), `osascript` на macOS, ввод в терминале на обычном Linux; при
  сбое любого диалога — откат в терминал. Сохраняй эти fallback'и.
- **Ресурс `pick_folder.ps1`** ищется через `importlib.resources.files("fs_tools.shared")`,
 НЕ вычислением пути от `__file__` (иначе не найдётся в установленном wheel). То же
 для `.env` (`shared/env.py`): путь через `FS_TOOLS_HOME`/CWD, не от `__file__`;
 загрузка — `load_dotenv(override=False)` (переменные процесса важнее значений файла).

## Синхронизация (`syncher`)

- **Однонаправленность** ПК → сервер; без offload-профиля локальные файлы не удаляются.
 Offload удаляет/архивирует только подтверждённо переданное; delete-guard блокирует
 массовые серверные удаления (код 3) до явного подтверждения.
- **Пути для rsync — posix** (`as_posix()`); источник всегда с завершающим `/`
 (содержимое каталога), чтобы поведение не зависело от слешей в конфиге.
- **Windows**: нативного `rsync` нет — работа через WSL (рекомендуется) или cwrsync;
 в WSL учитывай `/mnt/c/...`. SSH-цель (`host:path`) требует `ssh`; локальная
 (каталог→каталог) работает без него. Проверка бинарей на старте → код 1 при
 отсутствии (не трассировка).
- **Артефакты** (`.fs-syn.toml`, `.fs-log.log`, `.env`) не передаются никогда; журнал
 пишется в `production` и `dry-run` (в dry-run фиксируется последовательность
 событий, включая `(КОНФЛИКТ)`/`(ОШИБКА)`). Веб-хук `FSSYN_*`
 — только `https://` (токен не уходит по нешифрованному каналу).

## Осознанные допущения (не баги)

- **Зарезервированные имена Windows** (`CON`, `NUL`, `COM1..9`, `LPT1..9`) и
  превышение длины пути специально не обезвреживаются: такое `os.rename` падает,
 ошибка ловится, элемент **пропускается** (данные целы), фиксируется в
 `FsNormalizer.errlist` и даёт код возврата `2` — сигнал автоматизации.
- **Case-only** идёт в два шага через временное имя без отката: сбой второго шага
  крайне маловероятен (та же папка), объект остаётся под скрытым `.__normtmp_*`.
- На POSIX `os.rename` молча перезатирает `dest`; защита — предварительная проверка
  занятости. Конкурентный доступ к ФС вне модели угроз (один проход за запуск).
- Сбой записи `.fs-log.log` — вторичный артефакт: только предупреждение в `stderr`, на
  код возврата не влияет.
