# Импорты — по PEP 8 (isort), а не по использованию

> Claude-эквивалент [`.cursor/rules/imports.mdc`](../../.cursor/rules/imports.mdc). Применяется при работе с `**/*.py`.

Порядок импортов идиоматичный (PEP 8 / isort), не «по порядку использования». Блок
импортов — справочник, а не сценарий исполнения.

## Правила

- Группы (между группами — пустая строка), в порядке:
  1. `from __future__ import ...`
  2. стандартная библиотека
  3. сторонние пакеты
  4. локальные/относительные модули проекта (`from . import ...`, `from ..shared ...`)
- Внутри группы — по алфавиту; `import x` идёт перед `from x import y`.
- Исключение — ре-экспортирующие `__init__.py` (`fs_tools/normalizer/__init__.py`,
  `fs_tools/checker/__init__.py`, `rules/__init__.py`): порядок `__all__` отражает
  логику API/конвейера. Сами import-строки в них всё равно сортируются isort.
- **Ленивые импорты — намеренные**: тяжёлые/опциональные зависимости (`unidecode`,
  `requests`, `python-dotenv`) и runner режимов в диспетчере импортируются **внутри
  функций**, а не на уровне модуля (чтобы команда работала без чужого extra). Не
  поднимай их в шапку файла.

## Проверка (автоматическая)

Конвенция включена в `ruff` правилом `I` (isort) в `ruff.toml`:

    ruff check .                    # проверка (в т.ч. порядок импортов)
    ruff check --fix .              # автоисправление порядка

Не переставляй импорты вручную «по смыслу» — доверься `ruff --fix`.
