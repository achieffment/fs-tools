# Синхронизация правил Cursor ↔ Claude

Проект поддерживает оба редактора: [`.cursor/rules/*.mdc`](../../.cursor/rules/)
(Cursor) и [`.claude/rules/*.md`](../rules/) (Claude Code, точка входа —
[`CLAUDE.md`](../../CLAUDE.md)). Оба набора несут **один и тот же канон**
политик проекта; при правке любого правила в одном наборе — **в этом же
шаге** зеркально обновить парный файл в другом наборе.

## Механические различия формата (не считаются расхождением)

Ниже — единственные допустимые отличия между парными файлами; всё остальное
содержание (заголовки, порядок разделов, списки, примеры, формулировки
политик) должно быть идентично по смыслу.

| Аспект                              | Cursor `.mdc`                                    | Claude `.md`                                                                                                    |
|---------------------------------------|----------------------------------------------------|---------------------------------------------------------------------------------------------------------------------|
| Frontmatter                          | YAML `description` / `globs` / `alwaysApply`       | Отсутствует; вместо него первая строка после `# Заголовок` — `> Claude-эквивалент [.mdc-файл](../../.cursor/rules/<file>.mdc). Применяется <когда>.` |
| Расширение в кросс-ссылках           | `` `name.mdc` ``                                    | `` `name.md` ``                                                                                                      |
| Упоминания расширения в тексте       | `.mdc`                                              | `.md` или нейтральное «правила»                                                                                     |

Этот файл (`rules-sync.mdc`/`rules-sync.md`) — исключение из правила «Claude-версия
ссылается на Cursor-версию»: он описывает саму двухредакторную схему, поэтому
хатнота у Claude-версии нет — обе версии равноправно излагают канон.

## Порядок работы при изменении правила

1. Внести правку в исходный файл (`.mdc` или `.md`).
2. Сразу перенести правку в парный файл (см. карту), адаптировав только
   механические различия из таблицы выше.
3. При добавлении/удалении правила — обновить таблицу карты соответствия в
   этом файле и список правил в корневом [`CLAUDE.md`](../../CLAUDE.md).
4. Прогнать полный цикл проверок из
   [`audit-governor.md`](audit-governor.md) — оба каталога правил не входят
   в lint/test globs напрямую, но `AGENTS.md`/`README.md`/`CLAUDE.md` могут
   содержать ссылки на затронутые файлы.

## Антипаттерны

- Править `.mdc`, не перенеся эквивалентную правку в `.md` (и наоборот) в
  том же шаге, а не «потом».
- Копировать frontmatter Cursor в Claude-файл или наоборот.
- Оставлять расхождение в карте соответствия при добавлении нового правила.
- Считать расхождением естественную разницу в оформлении
  frontmatter/расширений — не переносить эти технические детали как «правку
  по смыслу».

## Проверка перед завершением задачи, затрагивающей правила

- [ ] Каждый изменённый `.mdc` имеет актуальную пару `.md` (и наоборот) —
      сверить по карте выше.
- [ ] Единственные отличия между парой — из таблицы «Механические различия».
- [ ] Карта соответствия актуальна.
- [ ] `AGENTS.md` и `CLAUDE.md` ссылаются на новое/изменённое правило, если
      оно публичное.
