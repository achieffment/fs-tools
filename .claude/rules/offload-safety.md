# Offload: безопасность локального удаления

> Claude-эквивалент [`.cursor/rules/offload-safety.mdc`](../../.cursor/rules/offload-safety.mdc). Применяется при работе с `src/fs_tools/syncher/*`.

Профиль `[[backup]]` выгружает каталог на сервер и при `after_push` ∈ {`delete`,
`archive`} убирает локальные файлы. Реализация — `syncher/offload.py`. Главный
инвариант: **локальный файл удаляется/переносится только после подтверждённой
передачи**; частичный успех не трогает непереданное.

## Поток (`run_offload`)

1. Передать каталог на сервер (по умолчанию без `--delete` — накопительный архив).
2. Сбой передачи (`rsync` вернул ненулевой код) → код 2, `after_push` НЕ выполняется.
3. `--dry-run` → только план передачи; локальные файлы не трогаются.
4. При `verify = true` повторный `rsync --dry-run --checksum` даёт множество
   «ещё к передаче» (`verify_pending`). Подтверждены = файлы области профиля
   (`source_files`) МИНУС это множество.
5. `after_push` применяется только к подтверждённым:
  - `delete` — удалить локальный файл, затем подчистить опустевшие каталоги (но не
     сам `local_root`); сохраняются только якорные include-каталоги (и уровни,
     явно заданные отдельным якорным include-паттерном); для одного якоря защищается
     ближайший уровень, вложенный одноимённый каталог не защищается автоматически;
  - `archive` — перенести в `archive_dir` (дефолт
    `<local_root>/../_fs-backup/<profile>/<YYYY-MM-DD>/`), сохраняя относительный путь;
   - `nothing` — ничего не делать.

## Область offload (`source_files`)

Берётся у самого rsync: `rsync --list-only` источника с теми же `--filter`-правилами
(правила профиля + безусловные артефакты), из листинга — только файлы. Это исключает
расхождение движков сопоставления (см. `rsync-mapping.md`): нельзя удалить локально
файл, который rsync исключил из передачи. Этот же список — область подтверждения для
`after_push`.

## Гарантии

- Никогда не удаляем непереданное (`verify`-подтверждение пофайловое).
- Ошибки отдельных файлов попадают в `errlist` и поднимают код профиля до 2, но не
  прерывают обработку остальных.
- `verify = false` подтверждает по факту успешной передачи (без повторной сверки) —
  допустимо только при осознанном выборе; дефолт `true`.

Меняя логику offload — синхронизируй `tests/syncher/test_offload.py`, README и
`examples/syncher/`.

Примечание: во внутренней модели допускается маппинг (`archive` → внутреннее действие
`backup`), но внешний TOML-контракт и поведение rsync не меняются.
