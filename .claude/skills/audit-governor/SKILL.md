# Audit Governor

## Режимы запуска

Используй только один из двух режимов:

- `audit changed` — аудит внесённых правок.
- `audit full` — полный аудит проекта.

Если режим не указан, по умолчанию используй `audit changed`.

Требование «до полного green» подразумевается для любого запроса аудита, даже
если пользователь не произнёс эту фразу явно — не повод уточнять или
останавливаться раньше.

## Обязательные источники истины

Перед аудитом прочитай:

- `AGENTS.md`
- `README.md`
- `examples/README.md`
- `examples/checker/README.md`
- `examples/normalizer/README.md`
- `examples/syncher/README.md`
- `examples/schemer/README.md`
- `pyproject.toml`
- `CLAUDE.md` (корневой) и все `.claude/rules/*.md`
- `checklist.md` этого skill — **обязателен к прочтению целиком** перед
  началом: единственный источник области аудита, критериев качества и
  команд проверки, не дублируется в этом файле.

## Порядок выполнения

1. Определи режим (`audit changed`/`audit full`) — см. выше.
2. По `checklist.md`: область аудита для выбранного режима, полный список
   обязательных проверок качества, обязательные команды.
3. Найди несоответствия и исправь их согласно «Политике исправлений» ниже.
4. Прогони обязательные команды из `checklist.md` до полного green.
5. Повтори аудит тем же режимом до отсутствия новых находок.

Если для прохождения проверок нужны изменения в `CLAUDE.md`, `.claude/rules/*.md`
или `AGENTS.md`, вноси минимальные согласованные правки и повторяй цикл проверок.

## Политика исправлений

- Исправляй автоматически всё, что можно исправить корректно и безопасно.
- Не вноси косметические изменения без пользы.
- Не ломай публичные контракты без крайней необходимости.
- Не ослабляй безопасность и кросс-платформенные гарантии.
- Не оставляй частичную проверку: завершай аудит только после валидации
  всех пластов и повторного цикла проверок.
- Если выявлено, что правила проекта или `AGENTS.md` недостаточны для
  прохождения проверок, обновляй их минимально и согласованно с фактическим
  поведением кода.

## Формат финального ответа

Возвращай кратко:

1. список изменений по файлам;
2. результаты обязательных проверок;
3. что исправлено для консистентности;
4. подтверждение DoD.
