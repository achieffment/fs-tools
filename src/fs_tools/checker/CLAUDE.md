# CLAUDE.md — `src/fs_tools/checker/`

Путь-локальные правила для режима `checker` (адаптировано из
`.cursor/rules/rule-matching.mdc` — область `rule.py`/`engine.py`). Общие
правила проекта — см. `../../../CLAUDE.md`.

## Семантика `.fs-check` и проверки

`.fs-check` — gitignore-подобный синтаксис с **инвертированным** смыслом:
строки задают пути, которые ОБЯЗАНЫ существовать. Разбор — `rule.py`,
проверка — `engine.py`.

**Парсинг (`rule.py`)**: кодировка `utf-8-sig`. Комментарий — только
**ведущий** `#` (инлайн `#` — литерал). Пустые строки игнорируются. Конечные
пробелы обрезаются, кроме экранированных `\ `. Разделитель сегментов —
POSIX `/`. Пропуск комментариев/пустых строк и классификацию (положительные /
`!`-негативы) делает сам `rule.py`; в `pathspec` уходят только шаблоны без
ведущего `!`. **Завершающий `/`** распознаётся ДО снятия якорного `/` и
делает мандат строго каталогом (`is_dir()`); без него — `exists()` (файл ИЛИ
папка). Нет `.fs-check` → `FsRuleError` (некорректный запуск, код возврата
1).

**Разворачивание (`engine.py`)**: правило = **префикс** (все сегменты, кроме
последнего — множество существующих якорей) + **последний сегмент** (мандат,
обязателен). Префикс разворачивается `pathlib.Path.glob`. Нарушение —
отсутствие мандата в найденном якоре. Если префикс не дал якорей — **не**
нарушение. **Пустой префикс** (правило из одного сегмента, напр.
`/Activities`) → единственный якорь = сам `root`. НЕ вызывать `root.glob(".")`
(ValueError на Python 3.13+). **Фиксированные промежуточные каталоги**
должны иметь собственные литеральные правила — иначе их отсутствие
маскирует более глубокие нарушения. Мандат сверяется **литерально** (без
glob в последнем сегменте). Скрытые каталоги (на `.`) не обходим; симлинки
не разыменовываем; нарушения дедуплицируются и сортируются.

**Негативы `!` (ordered pathspec)**: `!`-шаблоны компилируются в
`pathspec.PathSpec` (фабрика `_FACTORY` из
`fs_tools/shared/pathspec_compat.py`). Все строки `!` в `.fs-check` (после
снятия первого `!`) уходят в **единый ordered** pathspec-список; спец-веток
по типу паттерна нет. Матч выполняется по относительным путям якоря и
целевого мандата: `anchor` и `anchor/require`. Поддерживаются обычные
pathspec/gitignore-маски (`**`, `*`, `?`, `[]`) и порядок строк
(`last-match-wins` для конфликтующих паттернов). Короткая форма `!_Archive`
валидна как pathspec-шаблон basename и отсекает ветки `_Archive` на любой
глубине; явный эквивалент — `!**/_Archive/**`. В checker `!` трактуется
только как исключение: re-include не используется. Ведущие `!` схлопываются
(`!!/a` == `!/a`).
